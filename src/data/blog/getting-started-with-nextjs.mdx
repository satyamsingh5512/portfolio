# Getting Started with Next.js 15

Next.js 15 brings powerful new features that make building web applications faster and more intuitive than ever. In this guide, we'll explore the key concepts and get you up and running.

## What is Next.js?

Next.js is a React framework that provides a complete solution for building production-ready web applications. It offers:

- **Server-Side Rendering (SSR)** - Render pages on the server for better performance
- **Static Site Generation (SSG)** - Pre-render pages at build time
- **API Routes** - Build your backend API alongside your frontend
- **File-based Routing** - Automatic routing based on your file structure
- **Image Optimization** - Automatic image optimization and lazy loading

## The App Router

Next.js 15 uses the App Router, which introduces a new way to structure your application:

```typescript
// app/page.tsx
export default function HomePage() {
  return (
    <main>
      <h1>Welcome to Next.js 15</h1>
      <p>Building modern web apps has never been easier!</p>
    </main>
  );
}
```

## Server Components

By default, all components in the App Router are Server Components. This means they run on the server, reducing the JavaScript bundle sent to the client:

```typescript
// app/blog/page.tsx
async function BlogPage() {
  // This runs on the server
  const posts = await fetchPosts();
  
  return (
    <div>
      {posts.map(post => (
        <BlogCard key={post.id} post={post} />
      ))}
    </div>
  );
}
```

## Client Components

When you need interactivity, use the `'use client'` directive:

```typescript
'use client';

import { useState } from 'react';

export function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}
```

## Data Fetching

Next.js 15 makes data fetching simple and efficient:

```typescript
// Server Component - automatic caching
async function getData() {
  const res = await fetch('https://api.example.com/data', {
    cache: 'force-cache' // or 'no-store' for dynamic data
  });
  return res.json();
}

export default async function Page() {
  const data = await getData();
  return <div>{data.title}</div>;
}
```

## Styling Options

Next.js supports multiple styling approaches:

- **Tailwind CSS** - Utility-first CSS framework
- **CSS Modules** - Scoped CSS files
- **Global CSS** - Traditional stylesheets
- **CSS-in-JS** - Styled-components, Emotion, etc.

## Deployment

Deploy your Next.js app to Vercel with zero configuration:

```bash
npm run build
vercel deploy
```

Or deploy to any platform that supports Node.js.
